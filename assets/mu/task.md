- Default label config: `60d +12%/-6% drop-neutral`.
- This asset was scaffolded from the cross-asset first-round batch universe.
- MU is not currently blocked by missing code paths; it is blocked by unstable label/model behavior. The strongest lines so far either stay too positive-rate heavy or become badly validation-test asymmetric.
- MU now has multiple pass-level static candidates, so the next question is no longer "can MU pass the gate at all?" but "which candidate survives walk-forward and operator-rule pressure without collapsing into regime-specific behavior?"
- Current working split:
- `tb30 + ret_60 + vol_ratio_20_120` is the best balance-first model reference.
- `tb20 + ret_60 + vol_ratio_20_120 + top_11.5pct` is the best refreshed execution-style overlay candidate on the current data window.
- The currently wired live split is slightly more conservative than the pure research winner: execution now runs through `tb30 + ret_60 + vol_ratio_20_120 + top_12.5pct`, while the stronger `tb20` overlay remains a research-side comparison lane until it proves more stable.
- Final decision state:
- Live stays unchanged. Keep `tb30 + ret_60 + vol_ratio_20_120` as the balance-first live reference and keep the wired `tb30 + top_12.5pct` execution lane as the default monitor path.
- Research also stays clearly separated from live. `tb20 + ret_60 + vol_ratio_20_120 + top_11.5pct` is now the lead research-side execution rule, but it is still a `hold`, not a promotion, because the challenger family still does not beat the live default cleanly enough on forward robustness.
- The research hierarchy is now more specific than before. The main research challenger is the plain `tb20 + ret_60 + vol_ratio_20_120` family with a tighter `top_11.5pct` overlay. `drawdown_20 >= -0.25` remains the most credible secondary damage-guard probe, but it still sits below the plain challenger family because the later narrow guard scans showed only marginal incremental edge. Rearm-based variants and stacked drawdown-plus-rearm combos stay below both of those as research-only branches.
- The fresh cutoff sweep removed the old simplification idea. `tb20 + ret_60 + vol_ratio_20_120 + fixed_0.510` no longer behaves like a near-equivalent on this branch: it produced `0` held-out episodes and only `37.04%` forward hit rate with `-5.37%` mean average return, so the challenger should stay on percentile-based cutoffs rather than a fixed-score shortcut.
