# Ideas Pool

說明：

- 這份 `ideas.md` 用來存放「值得記住，但還沒排進本輪 `task.md`」的想法。
- 這裡不是正式待辦清單，不需要打勾。
- 若某個想法確定要做，請拆成可執行任務後移到 `task.md`。
- 若某個想法已經做過，正式結果請看 `results.tsv`，不要只留在這裡。

## 一、標記設計延伸

### 1. 改 barrier 比例

想法：

- 測 `+10% / -5%`
- 測 `+12% / -6%`
- 測 `+6% / -3%`

原因：

- 目前 `+8% / -4%` 只是第一版設定，不一定是最適合 GLD 的風險報酬比。

適合什麼時候做：

- 等 baseline 跑穩後
- 或目前 class balance 明顯不理想時

### 2. 改持有 horizon

想法：

- 測 `40d`
- 測 `80d`
- 測 `120d`

原因：

- 有些標的對中期趨勢的反應速度不同，`60d` 不一定是最有效窗口。

### 3. neutral 樣本其他處理方式

想法：

- 保留 neutral，改做 3-class
- keep-all binary
- 對 neutral 加較低權重

原因：

- 目前 `drop-neutral` 最乾淨，但可能損失資料量。

## 二、特徵工程延伸

### 4. 更長週期趨勢特徵

想法：

- `ret_40`
- `ret_60`
- `ret_120`
- `sma_gap_60`
- `sma_gap_120`

原因：

- 現在任務偏中期買點，可能需要更長週期資訊。

### 5. 回撤與反轉特徵

想法：

- `drawdown_40`
- `drawdown_60`
- 從最近低點反彈幅度
- 距離最近高點 / 低點的位置

原因：

- 若目標想更偏向「找低點」，回撤類特徵通常很關鍵。

### 6. 波動與壓縮特徵

想法：

- `volatility_20`
- `volatility_40`
- 波動收縮 / 擴張指標
- range compression 類特徵

原因：

- 中期突破或反轉，常和波動結構變化一起出現。

### 7. K 棒型態深化

想法：

- 長下影線
- 吞噬型態
- 連續紅黑 K 計數
- gap 後延續 / 反轉特徵

原因：

- 目前已經看到 `gap_up_flag` 類特徵有效，K 棒類訊號值得再深入。

## 三、模型方向延伸

### 8. 排序或回歸版本

想法：

- 預測 `future_return_60`
- 預測風險調整後報酬
- 把分類改成 ranking

原因：

- 若未來想做多檔資產比較，排序可能比二元分類更實用。

### 9. 更強 tabular 模型

想法：

- tree-based model
- gradient boosting
- light neural net

原因：

- logistic regression 很適合 baseline，但可能有線性上限。

注意：

- 只有在 baseline 特徵方向跑得差不多後，才值得切模型。

## 四、驗證與回測延伸

### 10. 更完整 walk-forward

想法：

- 更細時間切分
- 多輪滾動視窗
- 不同 regime 分開統計

原因：

- 中期策略很怕只在某段市場有效。

### 11. 簡單回測框架

想法：

- 預測為正才進場
- 固定持有 60 天或直到 barrier 命中
- 統計 CAGR、最大回撤、命中率

原因：

- 分類分數高，不代表真的適合交易。

### 12. threshold 轉交易規則

想法：

- 比較 threshold `0.45`、`0.50`、`0.55`
- 比較高信心樣本與全樣本

原因：

- 交易上不一定要吃全部訊號，可能只做最有把握的一小部分。

## 五、資料來源延伸

### 13. 加入外部市場特徵

想法：

- 美元指數
- 利率
- 黃金期貨
- VIX

原因：

- GLD 的中期表現可能受宏觀條件影響。

注意：

- 這會讓 repo 複雜度上升，先等純價量版本穩定。

### 14. 多資產泛化

想法：

- 同樣標記方法試在 `SPY`、`QQQ`、`TLT`

原因：

- 可以驗證這套 labeling 與特徵設計是不是只有 GLD 有效。

## 六、流程管理想法

### 15. 建立每輪研究摘要模板

想法：

- 每輪固定輸出：
  - 做了什麼
  - 最佳結果
  - 是否突破
  - 下輪建議

原因：

- 研究累積變多後，摘要比單看 `task.md` 更容易回顧。

### 16. 將大方向和本輪任務持續分離

想法：

- `ideas.md` 永遠只放候選方向
- `task.md` 永遠只放本輪 backlog

原因：

- 避免任務清單膨脹失真
- 也避免做過的東西和靈感池混在一起
